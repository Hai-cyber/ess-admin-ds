# Standard: Tenant UI Governance

## Purpose

Define what the tenant front admin can and cannot control.

## Allowed

- Staff roster management.
- Area and capacity adjustments.
- Business hour updates.
- Social links/public IDs.
- Upload and manage marketing media assets.

## Restricted

- Paid module activation unless explicitly delegated.
- Provider secret values.
- Cross-tenant data access.
- Operator billing controls.

## UX Requirements

- Each field group must display save scope (tenant, operator, restricted).
- Validation errors are explicit and actionable.
- Changes display last editor and update time.
